# 決策紀錄：多段落 `<i>` 的 HTML 結構處理

## 疑問

使用 `<i>` 跨多個段落時，輸出的 HTML 結構不一致：

```html
<!-- 第一段在 <p> 內部 -->
<p><i>　　第一段第一行。<br>
　　第一段第二行。<br>
<br></i></p>

<!-- 後續段落被 <i> 包在外面 -->
<i>
<p>　　第二段。<br>
<br></p>
</i>
```

看起來「怪怪的」，是否能讓所有段落統一成 `<i>` 包 `<p>` 的格式？

## 根本原因

這是 HTML 規格與 Markdown 處理器之間的結構衝突：

- `<i>` 是 **inline 元素**，規格上不能包含 **block 元素**（如 `<p>`）
- Markdown 處理器（remark）對每個段落自動生成 `<p>` 標籤
- `<i>` 若合併進第一行，就會落在 `<p>` 內部；後續段落的 `<p>` 由 remark 另外生成，`<i>` 無法跨越

## 為什麼不改

有一個替代方案：在預處理時偵測多段落 `<i>` 區塊，將整個區塊轉成帶有 `<p>` 標籤的 raw HTML，繞過 remark 的段落處理。這樣可以產出 `<i><p>...</p></i>` 的統一結構。

但此方案有兩個明顯缺點：

1. **`<i>` 內部的 Markdown 語法會失效**：轉為 raw HTML block 後，remark 不再處理裡面的內容，`**粗體**`、`` `code` `` 等語法會原樣輸出為文字。

2. **仍是非標準 HTML**：`<i>` 包 `<p>` 在 HTML5 規格中屬於無效結構（inline 包 block），不同渲染器行為不保證一致。

## 結論

實際測試確認：目前輸出雖然 HTML 結構不一致，但渲染器（編輯器）已正確將所有段落顯示為斜體，視覺效果符合預期。

在「HTML 結構統一但 Markdown 語法失效」與「HTML 結構不完美但功能完整」之間，選擇後者。**現狀即最佳解，不需修改。**
