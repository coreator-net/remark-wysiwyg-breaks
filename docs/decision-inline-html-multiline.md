# 決策紀錄：多段落 `<i>` 的 HTML 結構處理

## 疑問

使用 `<i>` 跨多個段落時，輸出的 HTML 結構不一致：

```html
<!-- 第一段在 <p> 內部 -->
<p><i>　　第一段第一行。<br>
　　第一段第二行。<br>
<br></i></p>

<!-- 後續段落被 <i> 包在外面 -->
<i>
<p>　　第二段。<br>
<br></p>
</i>
```

看起來「怪怪的」，是否能讓所有段落統一成 `<i>` 包 `<p>` 的格式？

## 根本原因

這是 HTML 規格與 Markdown 處理器之間的結構衝突：

- `<i>` 是 **inline 元素**，規格上不能包含 **block 元素**（如 `<p>`）
- Markdown 處理器（remark）對每個段落自動生成 `<p>` 標籤
- `<i>` 若合併進第一行，就會落在 `<p>` 內部；後續段落的 `<p>` 由 remark 另外生成，`<i>` 無法跨越

## 評估過的方案

### 方案一：整段轉為 raw HTML block（棄用）

在預處理時將整個 `<i>...</i>` 區塊轉成帶有 `<p>` 標籤的 raw HTML，產出 `<i><p>...</p></i>` 的統一結構。

缺點：
1. **`<i>` 內部的 Markdown 語法會失效**：remark 不再處理 raw HTML block 內的內容，`**粗體**`、`` `code` `` 等語法會原樣輸出為文字。
2. **仍是非標準 HTML**：`<i>` 包 `<p>` 在 HTML5 規格中屬於無效結構（inline 包 block）。

### 方案二：將標籤分配到每個段落（採用，v1.0.4）

在預處理階段偵測多段落 `<i>` 區塊，將 `<i>` 插入每個段落第一行的開頭、`</i>` 插入每個段落最後一行的結尾。

```
輸入：                     輸出：
<i>                →       （移除）
　　第一段第一行。           <i>　　第一段第一行。
　　第一段第二行。           　　第一段第二行。</i>
                            （空行）
　　第二段。</i>             <i>　　第二段。</i>
```

結果：每個段落都是合法的 `<p><i>…</i></p>`，Markdown 語法在各段落內繼續正常運作。

## 結論

v1.0.4 採用方案二，完整解決 HTML 結構不一致的問題，同時保留 Markdown 語法處理能力。**現狀（v1.0.4 之前）為過渡期產物，已修正。**
